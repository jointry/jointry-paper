\section{分析ワークシートのヴァリエーション}

\subsection*{プログラムの見える化}
\begin{itemize}
\item A「普通のプログラミングと同じようにできるから，考え方を学ぶにはすごくいいと思います」
\item C「できることの少なさが，正解を生み出しやすいと思う．キーボードだと，セミコロンの打ち間違いとか，あとは英語のスペルミスとか，全角で打っちゃうとか，そういうのでコンパイルエラーが起きちゃうので，それで動かないのがよくわからなくて，またそこで悩んだりして．ビジュアルプログラミングだと，そういうプログラムそのもので悩むことが少ないと思う」
\item A「（テキストだと）ハードル高いから，こっち（ビジュアルプログラミング）のほうがいいかもしれませんよね」
\end{itemize}

\subsection*{停止する思考}
\begin{itemize}
\item  B「ひとりでやってるときは考えているときに手が止まっちゃう．自分が思うようになかなかできない，手が出せない」
\item  C「ひとりでやっていると正しいかどうかがわからない」
\end{itemize}

\subsection*{身近なアドバイス}
\begin{itemize}
\item B「もっとヒントがほしいと思った．ブロックがどうやって動くのかを先にヒントとしてくれれば，もっとわかりやすかったと思う」
\item A「アドバイスを言ってもらったほうが助かる」
\item C「誰かに横について見てほしいという感じがする」
\end{itemize}

\subsection*{わかるけどわからない}
\begin{itemize}
\item A「動いているからわかりやすいけど，むずかしい」
\item C「ペンギンが歩いているのはわかるんですが，それをどうやって再現すればいいのかがよくわからない．そこはちょっと自分で考えないといけない，というのはあるかもしれない．」
\item A「いちおう動くんですけど，足踏みしてくれないこともあって……」
\end{itemize}

\subsection*{あ，ここ◯◯で出たところだ}
\begin{itemize}
\item A「これは……さっきのを繰り返せばいいんですかね」
\item B「これさっきと同じで，あと回転ですよね．壁まで行ったら戻るだけ」
\end{itemize}

\subsection*{わからないけどわかる不思議}
\begin{itemize}
\item B「動いてるのはわかるけど，この条件の意味がわかんないですよね．1 == 1ってどういう意味なんだろう」
\item A「これでいいんですかね．条件がわかってなくても」
\item C「条件がわかってなくても，動きを見たら正解だっていうのはわかるっていうのは，おもしろいですね」
\item A「なぜわかったか，不思議です（笑）」
\item A「わかんないけど，さわってれば，そのうちわかるかなあと思って」
\end{itemize}

\subsection*{楽しさ駆動}
\begin{itemize}
\item  A「やってみましょうよ．おもしろそうだから」
\item  C「わくわくするね」
\item  A「これ，楽しいですね．結構」
\item  B「うわ，止まっちゃったよ（笑）」
\item  C「これは，ダメですね（笑）」
\item  B「どんどんやっていくと面白いですね」
\end{itemize}

\subsection*{自発的な行動}
\begin{itemize}
\item A「［実行］ボタンを押してもいいですか？」
\item A「こっちでいじってもいいですか？」
\item B「実行してみますよ」
\item B「いったん（ブロックを）外しますね」
\item A「一回，やってみますよ」
\item A「じゃあ，触っていいですかね」
\item B「じゃあ，やりましょうか」
\item C「はい，実行しまーす」
\item B「ここに条件を入れたらいいんじゃない？」
\end{itemize}

\subsection*{共同プログラミング}
\begin{itemize}
\item C「みんなで同じものを共有できるって，いいですね」
\item B「チームで話しながらできるのは，いいですよね．同じ画面見ながらできるし」
\item A「ぼくは別に作りたいものがないんですよね．でもこれだと，みんなで一緒にやるから（いいと思う）」
\item C「話し合いながらできるっていいですよね．みんなで同じものを扱っているっていう．この感覚が」
\item B「課題を動く形で見せられたら，すごいなあって思うし．私も自分でもなんとか作ってみるかあって気持ちになる」
\item B「誰かがやってるのをちょっと見てたら，それで気付くところがあったら，手を出そうかなと思う．手が出しやすい」
\item B「もし同時に操作していると，名前が出ているとわかりやすい．誰が操作中か」
\end{itemize}

\subsection*{コールアンドレスポンス}
\begin{itemize}
\item A「こっちでいじってもいいですか？」C「どうぞどうぞ」
\item A「こっちでやっちゃっていいんですか？」C「どうぞ」
\item A「いっかいやってみましょうよ．おもしろそうだから」B「お願いします」
\item A「じゃあ，お願いします」C「はい，実行しまーす」
\item A「プラスでいいですか」C「プラスでいいと思いますよ」
\item C「条件のところに入ります？」A「あ，入りました」
\end{itemize}

\subsection*{伝えるチカラ}
\begin{itemize}
\item C「ひとりでやるプログラミングとは使う能力がぜんぜん違う感じがしました．これって，みんな同じで，グループで考えを共有したいときに，自分はこうだから，こうやってブロックを置きますよって，話し合う必要もあるので．あの，チームで何かモノを作る訓練にもなるかなあって．ひとりで作ってるんじゃないぞ，と」
\item A「就職試験のグループワークでよくやるやつみたい（笑）」
\item C「自分が考えているロジックを相手に伝えて，納得してもらってプログラミングをしないと，チームがバラバラになってしまう．なので，交渉能力というか，別の何かを使う気がします．プログラミングっていうのは，チームで組むことがメインじゃないですか．なので，これはこれで，いい教材だと思いますね」
\item A「チーム開発でコミュニケーション力がいるんで，できるひとが勝手にやったら，できない人は理解できないんで，自分がわかったところまでの内容をまわりに伝えられるか……．プログラム以外でも」
\end{itemize}

\subsection*{沈黙の合図}
\begin{itemize}
\item C「これは歩数を制御しなきゃいけないと思うんですよ．なので，変数を作るっていうのをやりたいと思います」A「……」B「……」A「変数ってラベルみたいなやつですよね」
\item C「たとえば，値が20より下だったら，この状態．20以下だったら，ずっと動いてますよね」B「……」A「……」B「変数に1を足すんですか？」
\item C「そうすると，歩数に1ずつたされるっていう」A「ん？」C「歩数に，歩数プラス1」A「……」B「……」A「ああ，わかった」
\end{itemize}

\subsection*{自分ならこうする}
\begin{itemize}
\item C「実行環境がひとつしかないと，取り合いになって，最後まで見たいのに，見れないっていうのがあるかも．試行錯誤用のパレットみたいなのがあると，またいいかもしれない」
\item A「操作を引き取る（独占する）機能があってもいいかもしれないですね」
\item C「ひとりのときは試行錯誤できるのに，オープンになると（共有されていると）やりにくい．でも，そういう部分も含めて，いい教育になりそう．みんなで試行錯誤したらいいと思います」
\end{itemize}
